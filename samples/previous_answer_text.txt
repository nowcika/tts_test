이전에 안 됐던 핵심 이유는 코드가 있었지만 실제 코지 보이스 실행 조건과 에이 피 아이 사용 방식이 안 맞았기 때문입니다.

구체적으로는 처음에는 코지 보이스 모델 로딩을 잘못 잡았습니다. 코지 보이스 투를 직접 쓰는 방식으로 접근했는데, 현재 구성에서는 공식 오토 모델 로더를 쓰는 게 맞았습니다.

기본 모델을 코지 보이스 투 영 점 오 비 쪽으로 생각했는데, 텍스트만 넣어서 바로 말하게 하는 에스 에프 티 방식에는 코지 보이스 삼백 엠 에스 에프 티가 더 맞았습니다.

기본 화자가 한국어가 아니었습니다. 모델 안에 있는 화자 목록을 확인해서 한국어 여성 화자를 기본값으로 바꿨습니다.

코지 보이스가 반환하는 오디오가 토치 텐서 형태였는데, 기존 저장 로직이 그걸 웨이브 파일로 제대로 변환하지 못했습니다. 이 부분을 넘파이 배열로 변환하고, 단일 채널 웨이브 형태로 정규화해서 저장하도록 고쳤습니다.

로컬 실행 환경도 문제였습니다. 파이썬 삼 점 십이에서는 코지 보이스 전체 설치 목록이 그대로 설치되지 않는 패키지가 있어서, 실제 실행에 필요한 패키지 조합으로 맞춰 설치했습니다.

그래서 지금은 다음 흐름으로 동작합니다.

첫째, 한국어 텍스트를 입력합니다.
둘째, 코지 보이스 삼백 엠 에스 에프 티 모델을 로드합니다.
셋째, 기본 한국어 화자를 선택합니다.
넷째, 코지 보이스가 음성 파형을 생성합니다.
다섯째, 파형을 스물두 점 영오 킬로헤르츠 모노 웨이브 파일로 저장합니다.
여섯째, 실제 생성된 웨이브 파일을 위스퍼로 다시 인식해서 한국어가 나오는지 확인합니다.

정상 동작을 위해 추가 학습이 꼭 필요한 건 아닙니다. 지금처럼 한국어 문장을 일반적인 한국어 음성으로 읽기 위한 목적이면, 사전 학습된 코지 보이스 모델만으로 됩니다.

다만 추가 학습이나 파인 튜닝이 필요한 경우는 있습니다.

특정 사람 목소리처럼 계속 말하게 만들고 싶을 때.
회사나 브랜드용 고정 화자 품질이 필요할 때.
긴 문장, 전문 용어, 억양, 감정 표현을 더 안정적으로 맞추고 싶을 때.
한국어 발음 품질을 실사용 수준으로 끌어올리고 싶을 때.

간단히 말하면, 지금 저장소에 올린 것은 학습 없이 로컬에서 한국어 텍스트를 웨이브 파일로 변환하는 도구입니다. 추가 학습 없이도 동작하지만, 특정 목소리나 더 높은 품질을 원하면 별도 음성 데이터로 파인 튜닝하거나 제로 샷 프롬프트 오디오 방식을 쓰는 쪽으로 가야 합니다.
